## Technical Production Notes

### Screen Recording Setup

**Browser Configuration**:
- Chrome Canary or Chrome Dev (latest version)
- Window size: 1920x1080 (full screen)
- Zoom: 100%
- Theme: Default light theme
- No extensions visible except FlashDoc
- Bookmarks bar: Hidden
- Developer tools: Closed

**Extension Configuration**:
- Load unpacked from source
- Configure all 5 slots with visible formats
- Create 3 presets: Writing, Coding, Research
- Set up 5-6 category shortcuts
- Enable Corner Ball

**Demo Content**:
- Use professional-looking webpage (Medium article, Dev.to post, or custom HTML)
- Clean typography
- Readable text (16-18px font size)
- Professional topic (tech, productivity, design)

**Recording Tool**:
- OBS Studio (free) or ScreenFlow (macOS)
- Settings:
  - 1920x1080 @ 60fps
  - Quality: High (CBR 40Mbps)
  - Format: MP4
  - Audio: 48kHz, 16-bit

---

### Editing Workflow

**Software Options**:
- DaVinci Resolve (free, professional)
- Adobe Premiere Pro
- Final Cut Pro (macOS)
- Kdenlive (free, Linux)

**Editing Steps**:
1. **Import** all screen recordings, logo assets, icon files
2. **Color Grading**: Apply LUTs for consistent look
   - Act 1: Cooler tones (Blue-Gray LUT)
   - Act 2-4: Warmer, saturated tones (Vibrant LUT)
3. **Speed Ramping**: Slow down key moments (button clicks), speed up transitions
4. **Annotations**: Add arrows, text overlays, highlights
   - Use After Effects or Motion for complex animations
   - Export as transparent .mov files, overlay in editor
5. **Audio Mixing**:
   - Music bed: -18dB baseline, peak at -12dB
   - Voiceover: -6dB average, normalize
   - SFX: -15dB, duck music during voiceover
6. **Captions**: Add subtitles for accessibility
   - White text, black background, 80% opacity
   - Bottom center, 2 lines max
7. **Color Correction**: Match all clips, add vignette
8. **Export**:
   - 1920x1080, 60fps
   - H.264, High Profile, 20Mbps
   - AAC audio, 320kbps

### Voiceover Recording

**Voice Talent**:
- Male or female, professional, friendly tone
- Age: 25-40 (relatable to target audience)
- Accent: Neutral (American, British, or Canadian)

**Recording Setup**:
- Quiet room, treated acoustics
- Condenser microphone (Rode NT-USB, Blue Yeti, or better)
- Pop filter
- Audio interface (if using XLR mic)

**Delivery Style**:
- Pace: Moderate (not rushed, not too slow)
- Tone: Confident but approachable
- Energy: Upbeat, friendly
- Emphasis: "FlashDoc", "Done", "Free"

**Recording Tips**:
- Record multiple takes
- Leave 2 seconds of silence at start/end (for noise removal)
- Deliver script twice: once natural, once with more energy (gives editing options)

**Post-Processing**:
- Noise reduction (-18dB)
- EQ: High-pass filter at 80Hz, slight boost at 3-5kHz (clarity)
- Compression: Ratio 3:1, threshold -12dB
- Normalize to -6dB
- De-esser if needed

### Accessibility

**Captions/Subtitles**:
- Embed in video (burned-in) or upload as .srt file to YouTube
- Font: Open Sans or Roboto, Bold
- Size: 48px (relative to 1080p)
- Color: White text, black background, 80% opacity
- Position: Bottom center
- Max 2 lines, 42 characters per line

**Audio Description** (optional, for full accessibility):
- Describe visual actions during pauses in voiceover
- Example: "A user selects text on a webpage. The FlashDoc menu appears."

**Color Contrast**:
- Ensure all text meets WCAG AA standards (4.5:1 ratio)
- Test with color blindness simulators
